<br>


## 2. 시나리오 (5~10줄)
이 프로그램은 배틀그라운드의 총기와 갑바 단계를 기준으로 적이 몇 발 맞으면 사망하는지를 계산하는 프로그램입니다.
1사용자는 총기 종류(AUG, 베릴, AKM, M416), 갑바 단계(0갑바, 1갑바,2갑바), 적 체력을 입력합니다.
시스템은 사용자가 선택한 총기에 맞는 기본 공격력을 내부에 자동으로 적용합니다.
또한 갑바 단계에 따라 감소된 피해량을 반영하여 실제 적용 공격력을 계산합니다.
이후 적 체력과 적용 공격력을 비교하여 사망까지 필요한 탄 수를 계산합니다.
계산이 완료되면 결과는 배열 형식으로 출력됩니다.
배열의 첫 번째 값은 총기, 두 번째 값은 갑바 단계, 세 번째 값은 적 체력, 네 번째 값은 사망 탄수입니다.
이를 통해 사용자는 총기와 갑바 조건에 따라 결과가 어떻게 달라지는지 확인할 수 있습니다

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
(예시)
1. 총기 선택
2. 갑바 단계 선택
3. 적 체력 입력
4. 사망 탄수 계산
5. 배열 형식 결과 출력

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
프로그램 개요: 배틀그라운드 총기 종류와 적의 갑바(방어구) 단계에 따라 사망까지 필요한 탄수를 계산하는 시스템을 구현했습니다. 사용자가 입력한 데이터를 리스트에 먼저 저장하고, 로직을 통해 도출된 결과를 해당 리스트에 추가하는 방식을 채택했습니다.

데이터 입력: input()으로 총기 이름, 적 체력, 갑바 단계를 입력받아 초기 리스트를 생성.

데미지 로직: 50m 몸샷 기준 가중치(감소율)를 적용하여 실제 공격력 산출.

0갑바(1.0), 1갑바(0.7), 2갑바(0.6), 3갑바(0.45)

사망 판정 로직: if-elif-else를 사용하여 총기별 기본 데미지를 할당하고, 갑바 단계에 따른 조건별 연산을 수행.

수학 연산: math.ceil() 함수를 활용하여 체력이 조금이라도 남으면 추가 탄환이 필요한 실전 상황 반영.
    
### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1: 배틀그라운드 TTK 로직 도출**
    * **프롬프트 요약:** "배그 총기(AUG, 베릴, AKM, M416)와 갑바 단계를 기준으로 적을 죽이는 데 필요한 탄수를 계산하는 프로그램을 만들 거야. 1학년 수준에서 리스트와 조건.문을 활용한 로직을 짜줘."
    * **적용 내용:** AI가 제안한 '갑바별 데미지 감소율'과 'math.ceil을 이용한 올림 계산' 방식을 적용하여 게임 메커니즘을 코드로 구현함.

2. **내용 2: 리스트 활용 및 구조 최적화**
    * **프롬프트 요약:** "처음 입력받은 값들을 리스트에 먼저 넣고, 나중에 계산된 탄수 결과만 그 리스트에 추가(append)하는 방식으로 수정해줘."
    * **적용 내용:** 단순 변수 나열을 넘어, result = [입력값들] -> result.append(결과값)으로 이어지는 리스트의 동적 활용법을 익힘.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 사망 탄수가 소수점으로 나오는 현상**
    * **원인:** 체력 / 데미지를 단순 계산하면 3.2발처럼 소수점이 발생함. 게임에서는 0.2발만 쏴도 1발을 소모해야 하므로 정수 출력이 필요했음.
    * **해결:** math.ceil() 함수를 사용하여 실전과 동일하게 '무조건 올림' 처리하여 해결함.
2. **문제 2: 다양한 자료형의 혼합 관리**
    * **원인:** 리스트 안에 문자열(총기명)과 실수(체력), 정수(탄수)가 섞여 있어 관리가 복잡해 보였음.
    * **해결:** 리스트는 서로 다른 자료형을 담을 수 있는 '바구니'라는 점을 이해하고, append()를 통해 순차적으로 데이터를 쌓아 결과 배열을 완성함.
